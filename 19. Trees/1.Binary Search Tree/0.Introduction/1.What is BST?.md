## 1. Introduction

A **Binary Search Tree (BST)** is a special type of Binary Tree where every node follows a specific ordering rule.

### BST Property

For every node:

```text
Left Subtree  <  Root  <  Right Subtree
```

or

```text
All values in left subtree are smaller than root.
All values in right subtree are greater than root.
```

Example:

```text
        50
       /  \
     30    70
    / \    / \
   20 40  60 80
```

* Left of 50 → 30, 20, 40 (all smaller)
* Right of 50 → 70, 60, 80 (all greater)

This property makes searching very efficient.

---

## 2. What is a Binary Search Tree?

A BST is a **hierarchical data structure** used for:

* Fast searching
* Fast insertion
* Fast deletion
* Maintaining sorted data

### Node Structure in Java

```java
class TreeNode {
    int val;
    TreeNode left;
    TreeNode right;

    TreeNode(int val) {
        this.val = val;
    }
}
```

## 3. Types of Binary Search Trees

### 1. Simple BST

Basic BST without balancing.

```text
      50
     /  \
   30    70
```

Operations depend on tree shape.

---

### 2. Balanced BST

Height remains approximately balanced.

Examples:

* AVL Tree
* Red-Black Tree

```text
        40
       /  \
      20   60
     / \   / \
    10 30 50 70
```

Search:

```text
O(log n)
```

---

### 3. Degenerate BST (Skewed BST)

Looks like a linked list.

```text
10
 \
 20
  \
  30
   \
    40
```

Search:

```text
O(n)
```

---

### 4. Self-Balancing BST

Automatically balances itself.

Examples:

* AVL Tree
* Red-Black Tree
* Splay Tree
* Treap

---

## 4. When to Use BST?

Use BST when:

### Fast Search Required

Example:

```text
Student Roll Numbers
Employee IDs
Product IDs
```

Searching:

```text
O(log n) (Balanced BST)
```

---

### Data Must Remain Sorted

Inorder Traversal of BST always gives:

```text
Sorted Order
```

Example:

```text
Input:
50,30,70,20,40

Inorder:
20 30 40 50 70
```

---

### Frequent Insertions and Deletions

BST supports:

```text
Insert → O(log n)
Delete → O(log n)
Search → O(log n)
```

(Balanced BST)

---

## 5. When Not to Use BST?

Avoid BST when:

### Data is Random and Unbalanced

Example:

```text
10
 \
 20
  \
  30
   \
    40
```

Performance becomes:

```text
O(n)
```

---

### Only Fast Search Needed

Use:

* HashMap
* HashSet

because:

```text
Search = O(1)
```

Average case.
