## 📌 Problem Statement

Given the head of a singly linked list, determine whether the linked list is a palindrome.

A linked list is a palindrome if it reads the same forward and backward.

---

## Example

### Input

```
1 → 2 → 2 → 1
```

### Output

```
true
```

Explanation:

Forward:

```
1 → 2 → 2 → 1
```

Backward:

```
1 → 2 → 2 → 1
```

Both are same, so it is a palindrome.

---

# 💡 Approach

We solve this problem using:

## Slow and Fast Pointer + Reverse Linked List

The algorithm has three steps:

---

# Step 1️⃣: Find Middle of Linked List

Use two pointers:

```
slow → moves one step

fast → moves two steps
```

Code:

```java
ListNode fast = head;
ListNode slow = head;

while(fast != null && fast.next != null){

    slow = slow.next;
    fast = fast.next.next;

}
```

When the loop ends:

```
slow = beginning of second half
```

---

# Dry Run: Finding Middle

Given:

```
1 → 2 → 2 → 1
```

Initial:

```
slow = 1
fast = 1
```

---

## Iteration 1

Move pointers:

```
slow = slow.next

slow = 2


fast = fast.next.next

fast = 2
```

Position:

```
1 → 2 → 2 → 1
    ↑    ↑
   slow fast
```

---

## Iteration 2

Move pointers:

```
slow = 2

fast = null
```

Position:

```
1 → 2 → 2 → 1
        ↑
       slow
```

Middle found.

The second half starts from:

```
2 → 1
```

---

# Step 2️⃣: Reverse Second Half

Reverse the linked list starting from `slow`.

Initial:

```
slow:

2 → 1 → null
```

Use three pointers:

```
prev
slow
next
```

---

## Iteration 1

Before:

```
prev = null

slow = 2 → 1
```

Execute:

```java
next = slow.next;
slow.next = prev;
prev = slow;
slow = next;
```

Changes:

```
next → 1

2 → null

prev → 2

slow → 1
```

Result:

```
Reversed part:

2 → null


Remaining:

1 → null
```

---

## Iteration 2

Before:

```
prev → 2

slow → 1
```

Execute:

```java
next = slow.next;
slow.next = prev;
prev = slow;
slow = next;
```

Changes:

```
next → null

1 → 2

prev → 1

slow → null
```

Final reversed second half:

```
1 → 2 → null
```

---

# Step 3️⃣: Compare Both Halves

Initialize:

```java
ListNode left = head;

ListNode right = prev;
```

Now:

```
Original:

1 → 2 → 2 → 1


Left:

1 → 2 → 2 → 1


Right:

1 → 2
```

Compare values one by one.

---

# Comparison Dry Run

## Iteration 1

```
left.val  = 1

right.val = 1
```

Equal ✅

Move:

```
left → 2

right → 2
```

---

## Iteration 2

```
left.val  = 2

right.val = 2
```

Equal ✅

Move:

```
left → 2

right → null
```

---

Loop condition:

```java
while(right != null)
```

fails because:

```
right == null
```

No mismatch found.

Therefore:

```
Palindrome = true
```

---

# ✅ Java Implementation

```java
class Solution {

    public boolean isPalindrome(ListNode head) {


        if(head == null || head.next == null){
            return true;
        }


        // Step 1: Find Middle

        ListNode fast = head;
        ListNode slow = head;


        while(fast != null && fast.next != null){

            slow = slow.next;
            fast = fast.next.next;

        }


        // Step 2: Reverse Second Half

        ListNode prev = null;


        while(slow != null){

            ListNode next = slow.next;

            slow.next = prev;

            prev = slow;

            slow = next;

        }


        // Step 3: Compare Two Halves

        ListNode left = head;

        ListNode right = prev;


        while(right != null){

            if(left.val != right.val){

                return false;

            }


            left = left.next;

            right = right.next;

        }


        return true;
    }
}
```

---

# 🧠 Algorithm Explanation

## Finding Middle

Fast pointer moves twice as fast as slow.

When fast reaches the end:

```
slow reaches middle
```

---

## Reversing Second Half

Example:

Before:

```
2 → 1
```

After:

```
1 → 2
```

Now we can compare:

```
First Half

1 → 2


Second Half

1 → 2
```

---

## Comparing Values

If any value differs:

```
return false
```

Otherwise:

```
return true
```
