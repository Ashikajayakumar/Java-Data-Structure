# Phase 1: Basic Hashing (Must Master)

## 1. Frequency Array

### Concept

Store how many times each element appears.

Example

```
Input:
1 2 2 3 3 3

Frequency

1 -> 1
2 -> 2
3 -> 3
```

### Used When

* Count frequency
* Counting characters
* Missing numbers
* Duplicates

### Time

```
O(N)
```

### LeetCode

* 136 Single Number
* 268 Missing Number
* 242 Valid Anagram
* 383 Ransom Note

---

## 2. HashMap

### Concept

Stores

```
Key -> Value
```

Example

```
apple -> 4
banana -> 2
orange -> 7
```

### Java

```java
HashMap<Integer,Integer> map = new HashMap<>();
```

### Operations

```
put()

get()

containsKey()

remove()

size()

isEmpty()
```

### Complexity

```
Insert  O(1)

Search  O(1)

Delete  O(1)
```

Average case.

### Questions

* Two Sum
* Subarray Sum Equals K
* Majority Element
* Isomorphic Strings
* Group Anagrams

---

## 3. HashSet

### Concept

Stores only unique elements.

Duplicates automatically removed.

```

Set

```

### Java

```java
HashSet<Integer> set = new HashSet<>();
```

Operations

```
add()

contains()

remove()
```

### Complexity

```
O(1)
```

Average

### Questions

* Contains Duplicate

* Happy Number

# Phase 2: Ordered Hashing

## 4. TreeMap

### Concept

Stores keys in sorted order.

Internally

```
Red Black Tree
```

Example

Input

```
5 2 7 1
```

TreeMap

```
1
2
5
7
```

### Complexity

```
Insert O(logN)

Search O(logN)

Delete O(logN)
```

### Used

* Sorted frequency
* Ceiling/Floor
* Range Queries

Questions

* Contains Nearby Almost Duplicate
* Skyline problems

# Phase 4: Advanced Hashing

## 8. Prefix Hashing

Idea

Instead of calculating every time

Precompute answers.

Example

```
Array

2 5 1 4 7
```

Prefix

```
2
7
8
12
19
```

Query

```
Sum(2,4)

=

prefix[4]-prefix[1]
```

instead of recalculating.

Applications

* Prefix Sum
* Prefix XOR
* Character frequency

Questions

* Range Sum Query
* Subarray Sum
* Count Nice Subarrays

---

## 9. Rolling Hash

Very important for Strings.

Instead of comparing

```
abcdef

abcxef
```

character by character,

convert into numbers.

Move window

```
abc

↓

bcd

↓

cde
```

Update hash in O(1).

Applications

* Duplicate substring
* Pattern matching

Questions

* Longest Duplicate Substring
* Repeated DNA Sequences

---

## 10. Rabin-Karp

Uses Rolling Hash.

Instead of checking every character

Compare hash.

```
Text

ABCDABCAB

Pattern

ABC
```

Compare

```
Hash(Text Window)

Hash(Pattern)
```

If same

Verify characters.

Complexity

Average

```
O(N)
```

Questions

* Implement strStr()
* Pattern Matching

---

## 11. Double Hashing

Problem

Different strings can have same hash.

Example

```
ABC

BAC
```

Sometimes collide.

Solution

Use

```
Hash1

Hash2
```

Collision probability becomes extremely small.

Used in

* Competitive Programming
* Advanced String Algorithms

---
