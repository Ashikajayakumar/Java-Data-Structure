## Problem Statement

A **peak element** is an element that is strictly greater than its neighbors.

Given an integer array `nums`, return the **index of any peak element**.

You may imagine that:

```text
nums[-1] = -∞
nums[n] = -∞
```

This means the first or last element can also be a peak.

You must solve it in **O(log n)** time.

---

## Example 1

**Input**

```text
nums = [1,2,3,1]
```

**Output**

```text
2
```

### Explanation

```text
Index : 0 1 2 3
Value : 1 2 3 1

Peak = 3
Index = 2
```

---

## Example 2

**Input**

```text
nums = [1,2,1,3,5,6,4]
```

**Output**

```text
5
```

### Explanation

```text
Index : 0 1 2 3 4 5 6
Value : 1 2 1 3 5 6 4
                    ^

Peak = 6
Index = 5
```

Another valid answer is index `1` because `2` is also a peak.

---

## Constraints

```text
1 <= nums.length <= 1000
-2^31 <= nums[i] <= 2^31 - 1
nums[i] != nums[i + 1]
```

## Java Solution

```java
class Solution {
    public int findPeakElement(int[] nums) {

        int left = 0;
        int right = nums.length - 1;

        while (left < right) {

            int mid = left + (right - left) / 2;

            if (nums[mid] < nums[mid + 1]) {
                left = mid + 1;
            } else {
                right = mid;
            }
        }

        return left;
    }
}
```
