# Container With Most Water (LeetCode 11)

## Problem

You are given an integer array `height`.

Each element represents the height of a vertical line.

Find two lines that together with the x-axis form a container, such that the container contains the **maximum amount of water**.

> **Return the maximum water that can be stored.**

---

## Example

```text
Input:
height = [1,8,6,2,5,4,8,3,7]

Output:
49
```

---

# Visualization

```text
height = [1,8,6,2,5,4,8,3,7]
index  =  0 1 2 3 4 5 6 7 8
```

---

# Formula

```
Area = width × height
```

Where

```
width = right - left

height = min(height[left], height[right])
```

Therefore,

```
Area = (right - left) × min(height[left], height[right])
```

---

# Brute Force Approach

## Idea

Check every possible pair of lines.

For every pair

- Calculate width
- Calculate minimum height
- Calculate area
- Update maximum area

---

## Algorithm

```
maxArea = 0

for i = 0 to n-1
    for j = i+1 to n-1

        width = j - i

        height = min(height[i], height[j])

        area = width × height

        maxArea = max(maxArea, area)

return maxArea
```

---

## Time Complexity

```
O(N²)
```

## Space Complexity

```
O(1)
```

---

# Why Brute Force is Slow?

For every element,

we compare with every other element.

```
Total comparisons

= n(n-1)/2

≈ O(N²)
```

For large arrays this becomes very slow.

We need a better solution.

---

# Optimal Approach (Two Pointers)

## Observation

Initially,

```
left = 0
right = n-1
```

This gives

- Maximum possible width

Now,

```
Area = width × minimum height
```

The width decreases every iteration.

Therefore,

the only way to get a larger area is by finding a taller minimum height.

---

# Key Idea

Move the pointer having the **smaller height**.

### Why?

Suppose

```
left height = 3

right height = 8
```

Current area

```
width × 3
```

Even if we move the taller line,

the minimum height is still

```
min(3, something)
```

Area cannot increase because

- width decreases
- limiting height remains 3

So moving the taller line never helps.

Instead,

move the smaller line hoping to find a taller one.

---

## Movement Rule

```
height[left] < height[right]
        left++

height[left] > height[right]
        right--

height[left] == height[right]
        move either pointer
```

---

# Algorithm

```
left = 0
right = n-1
maxArea = 0

while(left < right)

    width = right - left

    minHeight = min(height[left], height[right])

    area = width × minHeight

    maxArea = max(maxArea, area)

    if(height[left] < height[right])
        left++

    else
        right--

return maxArea
```

# Java Solution

```java
public class MaxAreaContainer {

    public static int maxArea(int[] height) {

        int left = 0;
        int right = height.length - 1;
        int maxArea = 0;

        while (left < right) {

            int width = right - left;
            int minHeight = Math.min(height[left], height[right]);
            int area = width * minHeight;

            maxArea = Math.max(maxArea, area);

            if (height[left] < height[right]) {
                left++;
            } else {
                right--;
            }
        }

        return maxArea;
    }

    public static void main(String[] args) {

        int[] height = {1,8,6,2,5,4,8,3,7};

        System.out.println(maxArea(height));
    }
}
```
