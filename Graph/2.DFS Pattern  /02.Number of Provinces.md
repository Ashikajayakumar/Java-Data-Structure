# LeetCode 547. Number of Provinces


# Problem
 
There are `n` cities.

You are given an `n x n` adjacency matrix `isConnected`.

* `isConnected[i][j] = 1` → city `i` and city `j` are directly connected.
* `isConnected[i][j] = 0` → no direct connection.

A **province** is a group of directly or indirectly connected cities.

Return the **number of provinces**.

---

## Example 1

```text
isConnected =

[
 [1,1,0],
 [1,1,0],
 [0,0,1]
]
```

Graph

```text
0 ----- 1

2
```

There are

* Province 1 → {0,1}
* Province 2 → {2}

Answer

```text
2
```

---

## Example 2

```text
[
 [1,0,0],
 [0,1,0],
 [0,0,1]
]
```

Graph

```text
0

1

2
```

Each city is isolated.

Answer

```text
3
```

---

# Idea

Think of each province as a **connected component**.

Algorithm:

* Start from every city.
* If it is not visited:

  * Found a new province.
  * DFS/BFS all cities connected to it.
* Count how many DFS calls are made.

---

# Algorithm

```text
visited[]

province = 0

for every city

      if not visited

            province++

            DFS(city)

return province
```

---

# Java Code (DFS)

```java
class Solution {

    public int findCircleNum(int[][] isConnected) {

        int n = isConnected.length;

        boolean[] visited = new boolean[n];

        int provinces = 0;

        for (int city = 0; city < n; city++) {

            if (!visited[city]) {

                provinces++;

                dfs(city, isConnected, visited);
            }
        }

        return provinces;
    }

    private void dfs(int city, int[][] graph, boolean[] visited) {

        visited[city] = true;

        for (int next = 0; next < graph.length; next++) {

            if (graph[city][next] == 1 && !visited[next]) {

                dfs(next, graph, visited);
            }
        }
    }
}
```

# Why do we increase `province` **before** DFS?

Suppose

```text
0 --- 1 --- 2
```

When loop reaches city 0

```text
province++

DFS(0)
```

DFS visits

```text
0
1
2
```

Now

```text
visited

T T T
```

The loop skips 1 and 2.

Only **one** DFS was needed.

Hence

```text
One DFS = One Province
```
