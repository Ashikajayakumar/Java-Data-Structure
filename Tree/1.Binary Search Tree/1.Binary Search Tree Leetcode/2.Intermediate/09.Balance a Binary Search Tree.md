# LeetCode 1382: Balance a Binary Search Tree

## Problem

Given a BST, return a **balanced BST** containing the same node values.

A balanced BST means:

```text
For every node:
|height(left) - height(right)| <= 1
```

---

## Example

### Input (Skewed BST)

```text
    1
     \
      2
       \
        3
         \
          4
```

This is a valid BST but not balanced.

### Output

```text
      2
     / \
    1   3
         \
          4
```

or

```text
      3
     / \
    2   4
   /
  1
```

Any balanced BST is accepted.

---

# Key Observation

### Inorder Traversal of BST

```text
BST → Inorder → Sorted Array
```

Example:

```text
    1
     \
      2
       \
        3
         \
          4
```

Inorder:

```text
[1, 2, 3, 4]
```

Now the problem becomes:

> Build a balanced BST from a sorted array.

This is exactly the same idea as **Convert Sorted Array to BST (LC 108)**.

---

# Approach

### Step 1

Store inorder traversal in a list.

```java
inorder(root, list);
```

Result:

```text
[1,2,3,4]
```

---

### Step 2

Choose middle element as root.

```text
mid = (left + right) / 2
```

This keeps the tree balanced.

---

### Step 3

Recursively build:

```text
Left BST  -> left half
Right BST -> right half
```

---

# Code

```java
class Solution {

    public TreeNode balanceBST(TreeNode root) {

        List<Integer> inorder = new ArrayList<>();

        inorder(root, inorder);

        return build(inorder, 0, inorder.size() - 1);
    }

    private void inorder(TreeNode root, List<Integer> list) {

        if (root == null) {
            return;
        }

        inorder(root.left, list);
        list.add(root.val);
        inorder(root.right, list);
    }

    private TreeNode build(List<Integer> list, int left, int right) {

        if (left > right) {
            return null;
        }

        int mid = left + (right - left) / 2;

        TreeNode root = new TreeNode(list.get(mid));

        root.left = build(list, left, mid - 1);
        root.right = build(list, mid + 1, right);

        return root;
    }
}
```

# Why Middle Element?

Suppose:

```text
[1,2,3,4,5,6,7]
```

Choose middle:

```text
4
```

Left half:

```text
[1,2,3]
```

Right half:

```text
[5,6,7]
```

Both sides have nearly equal nodes.

Thus:

```text
height(left) ≈ height(right)
```

which guarantees a balanced BST.
