# Kth Smallest Element in a BST (LeetCode 230)

## Key Observation

For a BST: 

```text
Inorder Traversal = Sorted Order
```

Because:

```text
Left → Root → Right
```

---

## Example

BST:

```text
        5
       / \
      3   6
     / \
    2   4
   /
  1
```

Inorder Traversal:

```text
1 2 3 4 5 6
```

So:

```text
1st smallest = 1
2nd smallest = 2
3rd smallest = 3
4th smallest = 4
```

---

# Simple Approach

Store inorder traversal in a list.

## Code

```java
class Solution {

    List<Integer> list = new ArrayList<>();

    public int kthSmallest(TreeNode root, int k) {

        inorder(root);

        return list.get(k - 1);
    }

    private void inorder(TreeNode root) {

        if (root == null) {
            return;
        }

        inorder(root.left);
        list.add(root.val);
        inorder(root.right);
    }
}
```

# Interview-Optimized Solution

Avoid storing all nodes.

Stop when the kth node is reached.

## Code

```java
class Solution {

    int count = 0;
    int ans = 0;

    public int kthSmallest(TreeNode root, int k) {

        inorder(root, k);

        return ans;
    }

    private void inorder(TreeNode root, int k) {

        if (root == null) {
            return;
        }

        inorder(root.left, k);

        count++;

        if (count == k) {
            ans = root.val;
            return;
        }

        inorder(root.right, k);
    }
}
```
