# LeetCode 173 — BST Iterator

## Problem

Design an iterator over a BST.

Implement:

```java
BSTIterator(TreeNode root)
int next()
boolean hasNext()
```

### Example

BST:

```text
        7
       / \
      3   15
         /  \
        9   20
```

Inorder Traversal:

```text
3 7 9 15 20
```

The iterator should return:

```text
3 → 7 → 9 → 15 → 20
```

one by one.

---

# Key Idea

We know:

```text
BST Inorder = Sorted Order
```

Instead of storing all values in a list, we use a **stack**.

The stack always stores the path to the next smallest node.

---

# Simple Interview Solution

```java
class BSTIterator {

    Stack<TreeNode> stack = new Stack<>();

    public BSTIterator(TreeNode root) {
        pushLeft(root);
    }

    public int next() {

        TreeNode node = stack.pop();

        if (node.right != null) {
            pushLeft(node.right);
        }

        return node.val;
    }

    public boolean hasNext() {
        return !stack.isEmpty();
    }

    private void pushLeft(TreeNode node) {

        while (node != null) {
            stack.push(node);
            node = node.left;
        }
    }
}
```

# Why pushLeft()?

```java
private void pushLeft(TreeNode node) {

    while (node != null) {
        stack.push(node);
        node = node.left;
    }
}
```

Example:

```text
        10
       /
      5
     /
    2
```

Push order:

```text
10
5
2
```

Stack:

```text
[10,5,2]
```

Top of stack:

```text
2
```

which is the smallest element.

# Complexity

### Constructor

```text
O(h)
```

where `h` = height of tree.

---

### next()

```text
Average O(1)
```

Each node is pushed and popped only once.

---

### hasNext()

```text
O(1)
```

---

### Space

```text
O(h)
```

Stack stores only one root-to-leaf path.

# Interview Trick

Remember:

```text
BST Iterator
=
Lazy Inorder Traversal
=
Stack + Push All Left Nodes
```

Core helper:

```java
while(node != null){
    stack.push(node);
    node = node.left;
}
```

This is the standard accepted solution for Binary Search Tree Iterator and is the version most commonly expected in interviews.
