
# Flip Equivalent Binary Trees (LeetCode 951)
<img width="1024" height="1536" alt="image" src="https://github.com/user-attachments/assets/54a53d3f-2bab-4d1c-b3f5-773738665ce9" />

## Problem

Two binary trees are **flip equivalent** if they are equal after performing any number of flip operations.

A **flip operation** swaps the left and right children of a node.

### Example

```
Tree 1:            Tree 2:

    1                  1
   / \                / \
  2   3              3   2
 /                    \
4                      4
```

Flip the children of node `1` in Tree 1:

```
    1
   / \
  3   2
       /
      4
```

Now both trees are the same.

---

## Recursive Idea

For every node:

### Case 1: No Flip

```
root1.left  ↔ root2.left
root1.right ↔ root2.right
```

### Case 2: Flip

```
root1.left  ↔ root2.right
root1.right ↔ root2.left
```

If either case is true, the trees are flip equivalent.

---

## Dry Run

```
root1 = [1,2,3]
root2 = [1,3,2]
```

At node `1`:

```
No Flip:
2 == 3 ❌

Flip:
2 == 2 ✔
3 == 3 ✔
```

Result = `true`

---

## Simple Java Code

```java
class Solution {
    public boolean flipEquiv(TreeNode root1, TreeNode root2) {

        if(root1 == null && root2 == null)
            return true;

        if(root1 == null || root2 == null)
            return false;

        if(root1.val != root2.val)
            return false;

        boolean noFlip =
                flipEquiv(root1.left, root2.left) &&
                flipEquiv(root1.right, root2.right);

        boolean flip =
                flipEquiv(root1.left, root2.right) &&
                flipEquiv(root1.right, root2.left);

        return noFlip || flip;
    }
}
```

:** Whenever you see *"compare two trees"* and *"structure can change by swapping children"*, think of checking **both possibilities (flip and no-flip)** recursively.
