# LeetCode 257: Binary Tree Paths

## Problem Statement

Given the root of a binary tree, return all root-to-leaf paths in any order.

A leaf is a node with no left child and no right child.

# Approach

Use **Depth First Search (DFS)**.

### Steps

1. Start from the root.
2. Add the current node value to the path.
3. If the node is a leaf:

   * Store the path in the answer list.
4. Otherwise:

   * Visit left subtree.
   * Visit right subtree.
5. Backtrack automatically by passing a new path string.


### Back to Root

Go Right

```text

```

Node 3 is a leaf.

```text
ans = ["1->2->5", "1->3"]
```

# Java Solution

```java
class Solution {

    public List<String> binaryTreePaths(TreeNode root) {

        List<String> result = new ArrayList<>();

        dfs(root, "", result);

        return result;
    }

    private void dfs(TreeNode node,
                     String path,
                     List<String> result) {

        if (node == null) {
            return;
        }

        if (path.length() == 0) {
            path = String.valueOf(node.val);
        } else {
            path = path + "->" + node.val;
        }

        if (node.left == null && node.right == null) {
            result.add(path);
            return;
        }

        dfs(node.left, path, result);
        dfs(node.right, path, result);
    }
}
```

# Interview Questions

### 1. Why use DFS?

Because we need to explore every root-to-leaf path completely before moving to another path.

---

### 2. When do we add a path to the result?

When the current node is a leaf:

```java
node.left == null && node.right == null
```

---

### 3. Time Complexity

```text
O(N)
```

Every node is visited once.

---

### 4. Space Complexity

```text
O(H)
```

Where `H` is the height of the tree (recursion stack).

* Balanced Tree: `O(log N)`
* Skewed Tree: `O(N)`

---

### Pattern

```text
Binary Tree Paths
        ↓
Root-to-Leaf Traversal
        ↓
DFS + Backtracking
```

This is one of the most common **root-to-leaf DFS pattern** questions asked before moving to problems like:

* Path Sum
* Path Sum II
* Sum Root to Leaf Numbers
* Pseudo-Palindromic Paths in a Binary Tree
