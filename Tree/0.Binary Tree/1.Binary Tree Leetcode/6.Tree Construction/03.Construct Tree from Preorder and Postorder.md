## Problem

Given:

* **Preorder** = Root → Left → Right
* **Postorder** = Left → Right → Root

---

## Example

```text
preorder  = [1,2,4,5,3,6,7]
postorder = [4,5,2,6,7,3,1]
```

Tree:

```text
        1
      /   \
     2     3
    / \   / \
   4  5  6  7
```





# Recursive Steps

1. Create root from preorder.
2. Next preorder element = left subtree root.
3. Find it in postorder.
4. Calculate left subtree size.
5. Recursively build:

   * Left subtree
   * Right subtree

---


---

# Easy-to-Understand Version

```java
class Solution {

    int preIndex = 0;

    public TreeNode constructFromPrePost(int[] preorder,
                                         int[] postorder) {

        return build(preorder, postorder,
                     0, postorder.length - 1);
    }

    private TreeNode build(int[] preorder,
                           int[] postorder,
                           int start,
                           int end) {

        TreeNode root =
                new TreeNode(preorder[preIndex++]);

        if (start == end)
            return root;

        int leftRoot = preorder[preIndex];

        int pos = start;

        while (postorder[pos] != leftRoot) {
            pos++;
        }

        root.left = build(preorder, postorder,
                          start, pos);

        root.right = build(preorder, postorder,
                           pos + 1, end - 1);

        return root;
    }
}
```

### Time Complexity

```text
O(n²)
```

### Step 2

```text

```

Find in postorder:

```text
[4,5,2,6,7,3,1]
      ^
```

Left subtree size = 3 nodes.

---

### Step 3

```text
        1
       /
      2
```

---

### Step 4

```text

       /
      2

    4   5
```

### Step 5

Build right subtree:

```text
        1
      /   \
     2     3
    / \   / \
   4  5  6  7
```

# Important Interview Note

### Unique Tree?

From:

```text
Preorder + Inorder
```

✅ Unique tree

From:

```text
Inorder + Postorder
```

✅ Unique tree

From:

```text
Preorder + Postorder
```

❌ Not always unique

Example:

```text
Preorder  = [1,2]
Postorder = [2,1]
```

Could be:

```text
1
/
2
```

or

```text
1
 \
  2
```

Both generate the same traversals.
