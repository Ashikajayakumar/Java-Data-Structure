# Minimum Depth of Binary Tree (LeetCode 111) 

## Problem

Find the minimum depth of a binary tree.

Minimum depth = Number of nodes along the shortest path from root node to nearest leaf node.

# Important Point

Minimum depth means:

```text id="0xpl9m"
Root → Nearest LEAF node
```

NOT nearest null node.

---

# Main Logic

1. If root is null → 0
2. If one child missing → go to other side
3. Otherwise:

   * find left depth
   * find right depth
   * take minimum + 1

---

# Formula

MinDepth(root)=1+\min(MinDepth(left),MinDepth(right))

---

# Easy Recursive Java Solution

```java id="7mql1p"
class Solution {

    public int minDepth(TreeNode root) {

        // Empty tree
        if(root == null){
            return 0;
        }

        // Left missing
        if(root.left == null){
            return minDepth(root.right) + 1;
        }

        // Right missing
        if(root.right == null){
            return minDepth(root.left) + 1;
        }

        // Both children exist
        int left = minDepth(root.left);
        int right = minDepth(root.right);

        return Math.min(left, right) + 1;
    }
}
```

---

# Why Special Conditions Needed?

Consider:

```text id="2mql8x"
    1
   /
  2
```

If we directly use:

```java id="6xpl4m"
Math.min(left, right) + 1
```

Then:

```text id="1xpl9m"
left = 1
right = 0

min = 0 + 1 = 1
```

Wrong answer.

Actual minimum depth:

```text id="8mql2p"
2
```

Because leaf node is `2`.
