# 🚀 Java Memory Management
<img width="1024" height="1536" alt="image" src="https://github.com/user-attachments/assets/aad8ab25-918e-4d83-875e-15eb148f8b9b" />

Java uses **JVM (Java Virtual Machine)** to manage memory automatically.

```text
+--------------------------------------------------------------+
|                        JVM MEMORY                            |
+------------------------------------+-------------------------+
|             HEAP SPACE             |       STACK MEMORY      |
|  (Shared among all threads)        |  (Private per thread)   |
+------------------------------------+-------------------------+
|  METASPACE (Class metadata)                                  |
+--------------------------------------------------------------+
```

---

# 1️⃣ Heap Memory

### What is Heap?

Heap stores **Objects and Instance Variables**.

```java
class Student {
    String name;
}

public class Main {
    public static void main(String[] args) {
        Student s = new Student();
    }
}
```

### Memory

```text
STACK                    HEAP
-----                    -----
s ---------------------> Student Object
                          name = null
```

### Features

✅ Shared among all threads

✅ Objects are stored here

✅ Garbage Collector cleans unused objects

---

# 2️⃣ Stack Memory

### What is Stack?

Stores:

* Local Variables
* Method Calls
* References

### Example

```java
public class Main {

    static void display() {
        int x = 10;
    }

    public static void main(String[] args) {
        display();
    }
}
```

### Memory

```text
TOP
+------------------+
| display() Frame  |
| x = 10           |
+------------------+
| main() Frame     |
+------------------+
BOTTOM
```

When `display()` finishes:

```text
display() frame removed
```

### Features

✅ Fast memory

✅ Each method gets a Stack Frame

✅ Automatically removed after method execution

---

# 3️⃣ Metaspace

### What is Metaspace?

Stores:

* Class Metadata
* Method Information
* Bytecode Information

### Example

```java
class Employee {
}
```

When JVM loads Employee class:

```text
Metaspace
---------
Employee Class Info
Methods Info
Fields Info
```

### Features

✅ Introduced in Java 8

✅ Replaced Permanent Generation (PermGen)

---

# 4️⃣ Young Generation

Part of Heap Memory.

Stores newly created objects.

```java
Student s = new Student();
```

New object goes to:

```text
Young Generation
   |
   +-- Eden Space
```

---

## Eden Space

All new objects start here.

```text
Young Generation
+---------+
|  Eden   |
+---------+
```

Example:

```java
new Student();
new Employee();
new String("Java");
```

Stored in Eden.

---

## Survivor Spaces (S0 and S1)

Objects that survive Garbage Collection move here.

```text
Young Generation
+------+------+
| S0   | S1   |
+------+------+
```

Example:

```java
Student s = new Student();
```

If object is still used:

```text
Eden → S0 → S1
```

---

# 5️⃣ Old Generation

Stores long-living objects.

```java
public class Main {

    static Student s = new Student();

    public static void main(String[] args) {

    }
}
```

Object remains alive for a long time.

```text
Young Gen
    ↓
Old Gen
```

### Features

✅ Long-lived objects

✅ Less frequent garbage collection

---

# 6️⃣ Garbage Collection (GC)

Java automatically removes unused objects.

### Example

```java
Student s = new Student();

s = null;
```

Now object has no reference.

```text
Student Object
     ↑
 No Reference
```

GC removes it.

---

# Complete Flow

```text
Object Created
      ↓
    Eden
      ↓
Survives GC
      ↓
     S0
      ↓
     S1
      ↓
Survives many GCs
      ↓
 Old Generation
      ↓
 No Reference
      ↓
 Garbage Collector Deletes
```


This is enough for most Java fresher interviews on JVM memory management.
