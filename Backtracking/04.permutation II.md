# LeetCode 47 - Permutations II (Step-by-Step for Beginners)

If you understood **Permutations (LeetCode 46)**, then **Permutations II** is very easy.

The only difference is:

* **Permutation I** → All numbers are unique.
* **Permutation II** → Numbers can have duplicates.

---

# Problem

Input

```text
nums = [1,1,2]
```

Output

```text
[
 [1,1,2],
 [1,2,1],
 [2,1,1]
]
```

Notice

```text
[1,1,2]
```

appears only **once**.

Without handling duplicates, the output would be

```text
[1,1,2]
[1,1,2] ❌ Duplicate
[1,2,1]
[1,2,1] ❌ Duplicate
[2,1,1]
[2,1,1] ❌ Duplicate
```

---

# Step 1 : Sort the Array

```java
Arrays.sort(nums);
```

Input

```text
[1,2,1]
```

becomes

```text
[1,1,2]
```

Why?

Because duplicate numbers become adjacent.

```text
1 1 2
↑ ↑
```

Now they are easy to detect.

---

# Step 2 : Think Like a Student

Three students

```text
1
1
2
```

Question

Arrange them in every possible order.

If we treat both **1's** as different students,

```text
1(first) 1(second) 2
```

and

```text
1(second) 1(first) 2
```

Both look exactly the same.

So we should generate only one.

---

# Code

```java
class Solution {

    List<List<Integer>> ans = new ArrayList<>();

    public List<List<Integer>> permuteUnique(int[] nums) {

        Arrays.sort(nums);

        boolean[] used = new boolean[nums.length];

        backtrack(nums, used, new ArrayList<>());

        return ans;
    }

    void backtrack(int[] nums, boolean[] used, List<Integer> curr) {

        if (curr.size() == nums.length) {
            ans.add(new ArrayList<>(curr));
            return;
        }

        for (int i = 0; i < nums.length; i++) {

            if (used[i])
                continue;

            // Skip duplicates
            if (i > 0 && nums[i] == nums[i - 1] && !used[i - 1])
                continue;

            // Choose
            used[i] = true;
            curr.add(nums[i]);

            // Explore
            backtrack(nums, used, curr);

            // Undo
            curr.remove(curr.size() - 1);
            used[i] = false;
        }
    }
}
```

# Why Skip?

Imagine two identical twins.

```text
1(first)

1(second)
```

If we start the permutation with

```text
1(first)
```

we get

```text
1 1 2

1 2 1
```

If we instead start with

```text
1(second)
```

we get exactly the same permutations again.

So we skip the second twin **until the first twin has been used**.

# Why `!used[i-1]`?

This is the trick that beginners usually struggle with.

Suppose

```text
used

F F F
```

You are trying to choose

```text
second 1
```

while

```text
first 1
```

has not been chosen.

That means

```text
Second twin came before first twin.
```

This creates duplicate permutations.

So skip it.

---

Now suppose

```text
used

T F F
```

Meaning

```text
First 1 already selected.
```

Now choosing

```text
second 1
```

is perfectly valid.

Because the order becomes

```text
1(first)

↓

1(second)
```

No duplicate is produced.

---

# Real-Life Example

Imagine two identical red balls.

```text
🔴
🔴
⚪
```

Can you tell which red ball is first?

No.

So

```text
🔴 🔴 ⚪
```

and

```text
🔴 🔴 ⚪
```

are exactly the same arrangement.

Our condition simply avoids generating the second arrangement.
