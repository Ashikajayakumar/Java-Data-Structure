This is the **first and most important Backtracking problem**. If you understand this problem well, you'll find it much easier to solve Permutations, Combination Sum, N-Queens, and other backtracking problems.

---

# Problem

Given an integer array `nums` of **unique** elements, return **all possible subsets** (the power set).

The order of the subsets does not matter.

### Example

Input

```text
nums = [1,2,3]
```

Output

```text
[]
[1]
[2]
[3]
[1,2]
[1,3]
[2,3]
[1,2,3]
```

Notice there are **8 subsets**.

Why?

Because each element has **2 choices**:

* Take it
* Don't take it

For 3 elements:

```text
2 × 2 × 2 = 8
```

# How Backtracking Thinks

Start with an empty subset.

```text
[]
```

Should we take **1**?

Choice 1

```text
Take 1
```

Choice 2

```text
Skip 1
```

Then do the same for **2**.

Then **3**.

---

## Decision Tree

```
                         []
                    /            \
               Take1             Skip1
              /                     \
          [1]                        []
         /   \                    /      \
     Take2   Skip2           Take2      Skip2
      / \      / \            / \         / \
```

Each path from the root to a leaf is one subset.

---

# Another Way (Used in LeetCode)

Instead of explicitly writing "take" and "skip", we use a loop.

At every recursion:

* Save the current subset.
* Try adding each remaining element one by one.
* Recurse.
* Remove the last element (backtrack).

# Complete Recursion Tree

```
                         []
              /------------|------------\
             1             2             3
            /             /             /
         [1]           [2]           [3]
        /   \            \              \
     [1,2] [1,3]       [2,3]        (end)
        |
     [1,2,3]
```

Every node in this tree is a valid subset, so we save it as soon as we visit it.

# Java Code

```java
class Solution {

    List<List<Integer>> ans = new ArrayList<>();

    public List<List<Integer>> subsets(int[] nums) {

        backtrack(0, nums, new ArrayList<>());

        return ans;
    }

    private void backtrack(int start, int[] nums, List<Integer> subset) {

        ans.add(new ArrayList<>(subset));

        for (int i = start; i < nums.length; i++) {

            // Choose
            subset.add(nums[i]);

            // Explore
            backtrack(i + 1, nums, subset);

            // Undo
            subset.remove(subset.size() - 1);
        }
    }
}
```

---

# Line-by-Line Explanation

### Save the current subset

```java
ans.add(new ArrayList<>(subset));
```

Even an empty subset is a valid answer, so we save it immediately.

---

### Loop through remaining elements

```java
for(int i = start; i < nums.length; i++)
```

We only consider elements from `start` onward to avoid duplicates and preserve order.

---

### Choose

```java
subset.add(nums[i]);
```

Include the current number.

Example:

```text
[]
↓

[1]
```

---

### Explore

```java
backtrack(i + 1, nums, subset);
```

Move to the next position.

If we picked index `i`, the next choice starts from `i + 1`.

---

### Undo (Backtrack)

```java
subset.remove(subset.size() - 1);
```

Remove the last chosen element so we can try the next possibility.

Example:

```text
[1,2]

↓

remove 2

↓

[1]
```

Without this undo step, the subset would keep growing incorrectly.

---

# Why `i + 1`?

Suppose:

```text
nums = [1,2,3]
```

After choosing `1`, we only want to consider:

```text
2
3
```

We don't want to choose `1` again, so the next recursion starts from the next index.
