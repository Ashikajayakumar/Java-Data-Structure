# LeetCode 90 - Subsets II (Step-by-Step for Beginners)


This problem is almost the same as **Subsets**, but there is one extra challenge:

The input array **contains duplicate numbers**, and the output **must not contain duplicate subsets**.

---

# Problem

Input

```text
nums = [1,2,2]
```

Output

```text
[]
[1]
[1,2]
[1,2,2]
[2]
[2,2]
```

Notice

```text
[2]
```

appears only **once**.

Without handling duplicates, we would get

```text
[]
[1]
[1,2]
[1,2,2]
[1,2]
❌ duplicate

[2]
[2,2]
[2]
❌ duplicate
```

---

# Step 1: Sort the Array

Before doing anything, sort it.

```java
Arrays.sort(nums);
```

Input

```text
[2,1,2]
```

becomes

```text
[1,2,2]
```

Why?

Because duplicate numbers become adjacent.

```text
1 2 2
    ↑↑
```

Now they are easy to detect.

# Code

```java
class Solution {

    List<List<Integer>> ans = new ArrayList<>();

    public List<List<Integer>> subsetsWithDup(int[] nums) {

        Arrays.sort(nums);

        backtrack(0, nums, new ArrayList<>());

        return ans;
    }

    void backtrack(int start, int[] nums, List<Integer> subset) {

        ans.add(new ArrayList<>(subset));

        for (int i = start; i < nums.length; i++) {

            if (i > start && nums[i] == nums[i - 1])
                continue;

            subset.add(nums[i]);

            backtrack(i + 1, nums, subset);

            subset.remove(subset.size() - 1);
        }
    }
}
```

# The Most Important Line

```java
if (i > start && nums[i] == nums[i - 1])
    continue;
```

Let's break it down:

### `i > start`

This means **we are not looking at the first choice in the current recursive call**.

### `nums[i] == nums[i - 1]`

The current number is the same as the previous number.

### `continue`

Skip it because choosing this duplicate **at the same recursion level** would generate the same subsets again.
