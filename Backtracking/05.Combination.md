# LeetCode 77 - Combinations


Many beginners confuse **Subsets** and **Combinations** because both use a `start` index.

Let's understand it from the beginning.

---

# What is a Combination?

A **combination** means:

> Choose **exactly k elements** from n elements.

The **order does not matter**.

Example

```text
n = 4
k = 2
```

Numbers

```text
1 2 3 4
```

Possible answers

```text
[1,2]
[1,3]
[1,4]
[2,3]
[2,4]
[3,4]
```

Notice

```text
[1,2]
```

and

```text
[2,1]
```

are the **same combination**.

---

# Real-Life Example

Suppose your teacher says:

> Select **2 students** from 4 students to participate in a competition.

Students

```text
A
B
C
D
```

Possible selections

```text
A B
A C
A D
B C
B D
C D
```

Notice

```text
A B
```

is the same as

```text
B A
```

because we are only selecting students, **not arranging them**.

# Difference Between Subsets and Combinations

Input

```text
1 2 3
```

### Subsets

Choose any number of elements.

```text
[]
[1]
[2]
[3]
[1,2]
[1,3]
[2,3]
[1,2,3]
```

---

### Combinations

Choose **exactly k elements**.

Suppose

```text
k = 2
```

Answer

```text
[1,2]
[1,3]
[2,3]
```

Nothing else.

# Decision Tree

```text
                 []
          /      |      |      \
         1       2      3       4
       / | \     | \
      2 3 4      3 4
```

Only the paths with **2 numbers** are answers.

---

# Java Code

```java
class Solution {

    List<List<Integer>> ans = new ArrayList<>();

    public List<List<Integer>> combine(int n, int k) {

        backtrack(1, n, k, new ArrayList<>());

        return ans;
    }

    void backtrack(int start, int n, int k, List<Integer> curr) {

        if (curr.size() == k) {
            ans.add(new ArrayList<>(curr));
            return;
        }

        for (int i = start; i <= n; i++) {

            // Choose
            curr.add(i);

            // Explore
            backtrack(i + 1, n, k, curr);

            // Undo
            curr.remove(curr.size() - 1);
        }
    }
}
```

# Why `start`?

Suppose we choose

```text
1
```

The next number should only come from

```text
2
3
4
```

We should never go back to

```text
1
```

or

```text
2
1
```

Otherwise, we'd generate duplicates like:

```text
[1,2]
[2,1] ❌
```

That's why we call:

```java
backtrack(i + 1, n, k, curr);
```

---

# Choose → Explore → Undo

Every iteration follows the same pattern:

### Choose

```java
curr.add(i);
```

Example

```text
[]

↓

[2]
```

---

### Explore

```java
backtrack(i + 1, n, k, curr);
```

Build the rest of the combination.

---

### Undo

```java
curr.remove(curr.size() - 1);
```

Example

```text
[2,4]

↓

remove 4

↓

[2]
```

Now we can try the next number.

---

# Time Complexity

There are exactly

```text
C(n,k)
```

possible combinations.

Each answer takes `O(k)` to copy.

Overall complexity is approximately:

```text
O(C(n,k) × k)
```
