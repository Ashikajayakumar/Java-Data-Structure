## 📌 What is Time Complexity?

Time complexity is the amount of time an algorithm takes to run as a function of input size **n**.

👉 It helps us compare algorithms and choose the most efficient one.

---

## 📊 Types of Time Complexity Notation

### 1️⃣ Big-O Notation (O) → Worst Case

* Represents the **maximum time** taken
* Most commonly used in interviews

📌 Example:

* Linear Search → **O(n)**

---

### 2️⃣ Omega Notation (Ω) → Best Case

* Represents the **minimum time** taken
* Rarely used in practice

📌 Example:

* Linear Search best case → **Ω(1)** (element found at first index)

---

### 3️⃣ Theta Notation (Θ) → Average Case

* Represents **average performance**
* Tight bound (best + worst balanced)

📌 Example:

* Binary Search → **Θ(log n)**

---

## ⚡ Best, Average, Worst Case

| Case         | Meaning             | Notation |
| ------------ | ------------------- | -------- |
| Best Case    | Minimum operations  | Ω(n)     |
| Average Case | Expected operations | Θ(n)     |
| Worst Case   | Maximum operations  | O(n)     |

# 🚀 Time Complexity with Java Examples

---

## 1️⃣ O(1) → Constant Time

* Execution time does not depend on input size

📌 Example:

```java
public class ConstantTime {
    public static void main(String[] args) {
        int[] arr = {10, 20, 30, 40};

        // Always same time operation
        System.out.println(arr[0]); // O(1)
    }
}
```

---

## 2️⃣ O(log n) → Logarithmic Time

* Problem size reduces in each step

📌 Example: Binary Search

```java
public class BinarySearch {
    public static int binarySearch(int[] arr, int key) {
        int low = 0, high = arr.length - 1;

        while (low <= high) {
            int mid = (low + high) / 2;

            if (arr[mid] == key)
                return mid;
            else if (arr[mid] < key)
                low = mid + 1;
            else
                high = mid - 1;
        }
        return -1;
    }
}
```

---

## 3️⃣ O(n) → Linear Time

* One full pass through data

📌 Example: Linear Search

```java
public class LinearSearch {
    public static int search(int[] arr, int key) {
        for (int i = 0; i < arr.length; i++) {
            if (arr[i] == key)
                return i;
        }
        return -1;
    }
}
```

## 4️⃣ O(n log n) → Linearithmic Time

* Divide and conquer algorithms

📌 Example: Merge Sort

```java
public class MergeSort {

    public static void mergeSort(int[] arr, int left, int right) {
        if (left < right) {
            int mid = (left + right) / 2;

            mergeSort(arr, left, mid);
            mergeSort(arr, mid + 1, right);

            merge(arr, left, mid, right);
        }
    }

    public static void merge(int[] arr, int left, int mid, int right) {
        // simple merge logic (short version)
        int[] temp = new int[right - left + 1];
        int i = left, j = mid + 1, k = 0;

        while (i <= mid && j <= right) {
            if (arr[i] <= arr[j])
                temp[k++] = arr[i++];
            else
                temp[k++] = arr[j++];
        }

        while (i <= mid) temp[k++] = arr[i++];
        while (j <= right) temp[k++] = arr[j++];

        for (i = 0; i < temp.length; i++)
            arr[left + i] = temp[i];
    }
}
```

## 5️⃣ O(n²) → Quadratic Time

* Nested loops

📌 Example:

```java
public class Quadratic {
    public static void main(String[] args) {
        int n = 5;

        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                System.out.print("(" + i + "," + j + ") ");
            }
            System.out.println();
        }
    }
}
```

---

## 6️⃣ O(2ⁿ) → Exponential Time

* Very slow, recursion-heavy problems

📌 Example: Fibonacci (naive recursion)

```java
public class Fibonacci {
    public static int fib(int n) {
        if (n <= 1)
            return n;

        return fib(n - 1) + fib(n - 2);
    }

    public static void main(String[] args) {
        System.out.println(fib(5));
    }
}
```
