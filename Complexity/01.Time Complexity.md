
# 🚀 Time Complexity in Data Structures & Algorithms (DSA)

## 📌 What is Time Complexity?

Time complexity is the amount of time an algorithm takes to run as a function of input size **n**.

👉 It helps us compare algorithms and choose the most efficient one.

---

## 📊 Types of Time Complexity Notation

### 1️⃣ Big-O Notation (O) → Worst Case

* Represents the **maximum time** taken
* Most commonly used in interviews

📌 Example:

* Linear Search → **O(n)**

---

### 2️⃣ Omega Notation (Ω) → Best Case

* Represents the **minimum time** taken
* Rarely used in practice

📌 Example:

* Linear Search best case → **Ω(1)** (element found at first index)

---

### 3️⃣ Theta Notation (Θ) → Average Case

* Represents **average performance**
* Tight bound (best + worst balanced)

📌 Example:

* Binary Search → **Θ(log n)**

---

## ⚡ Best, Average, Worst Case

| Case         | Meaning             | Notation |
| ------------ | ------------------- | -------- |
| Best Case    | Minimum operations  | Ω(n)     |
| Average Case | Expected operations | Θ(n)     |
| Worst Case   | Maximum operations  | O(n)     |

---

# 📦 Big-O Notation (6 Common Types)

## 1️⃣ O(1) → Constant Time

* Execution time does not depend on input size

📌 Example:

```java
arr[0]
```

---

## 2️⃣ O(log n) → Logarithmic Time

* Problem size reduces in each step

📌 Example:

* Binary Search

---

## 3️⃣ O(n) → Linear Time

* One full pass through data

📌 Example:

* Linear Search

---

## 4️⃣ O(n log n) → Linearithmic Time

* Divide and conquer algorithms

📌 Example:

* Merge Sort, Quick Sort

---

## 5️⃣ O(n²) → Quadratic Time

* Nested loops

📌 Example:

```java
for i in n:
   for j in n:
```

---

## 6️⃣ O(2ⁿ) / O(n!) → Exponential / Factorial Time

* Very slow, used in recursion/backtracking
