# LeetCode 152: Maximum Product Subarray

## Problem Statement

Given an integer array `nums`, find the contiguous subarray that has the largest product and return the product.

A **subarray** is a contiguous non-empty sequence of elements within an array.

---

## Example 1

```java
Input: nums = [2,3,-2,4]

Output: 6
```

### Explanation

Subarray:

```text
[2,3]
```

Product:

2\times3=6

Output:

```text
6
```

---

## Example 2

```java
Input: nums = [-2,0,-1]

Output: 0
```

### Explanation

Possible products:

```text
[-2]  = -2
[0]   = 0
[-1]  = -1
```

Maximum product:

```text
0
```

---

# Why This Problem Is Tricky

In **Maximum Sum Subarray (Kadane's Algorithm)**, we only track one value.

For products:

```text
Negative × Negative = Positive
```

A very small negative product can suddenly become the largest positive product later.

Therefore we must track:

* Maximum product ending at current position
* Minimum product ending at current position

---

## Example

```java
nums = [2,3,-2,4]
```

At:

```text
2 × 3 = 6
```

When we see:

```text
-2
```

Current maximum becomes:

```text
6 × (-2) = -12
```

But later another negative number may convert:

```text
-12 × (-5) = 60
```

into a huge positive product.

That's why we keep both max and min.

---

# Brute Force Approach

## Idea

Generate every subarray.

Calculate its product.

Track the maximum.

---

## Java Solution (Brute Force)

```java
class Solution {
    public int maxProduct(int[] nums) {

        int answer = Integer.MIN_VALUE;

        for (int i = 0; i < nums.length; i++) {

            int product = 1;

            for (int j = i; j < nums.length; j++) {

                product *= nums[j];

                answer = Math.max(answer, product);
            }
        }

        return answer;
    }
}
```



# Optimal Approach (Dynamic Programming)

## Key Idea

For every element:

Maintain:

```text
maxProductEndingHere
minProductEndingHere
```

Because:

```text
Negative × Negative = Positive
```

The smallest product can become the largest later.

---

## Transition

For current number:

```text
num
```

Possible new maximum:

```text
num

num × previousMax

num × previousMin
```

Take the largest.

Possible new minimum:

```text
num

num × previousMax

num × previousMin
```

Take the smallest.

# Java Solution (Optimal)

```java
class Solution {
    public int maxProduct(int[] nums) {

        int maxProd = nums[0];
        int minProd = nums[0];
        int answer = nums[0];

        for (int i = 1; i < nums.length; i++) {

            int num = nums[i];

            int tempMax = maxProd;

            maxProd = Math.max(
                    num,
                    Math.max(num * maxProd,
                             num * minProd)
            );

            minProd = Math.min(
                    num,
                    Math.min(num * tempMax,
                             num * minProd)
            );

            answer = Math.max(answer, maxProd);
        }

        return answer;
    }
}
```


**Time Complexity:** O(n)
**Space Complexity:** O(1)**
