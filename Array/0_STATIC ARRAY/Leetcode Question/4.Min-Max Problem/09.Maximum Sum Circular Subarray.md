# LeetCode 918: Maximum Sum Circular Subarray

## Problem Statement

Given a **circular integer array** `nums`, return the maximum possible sum of a non-empty subarray.

A circular array means:

```text
The end of the array connects to the beginning.
```

So a subarray may wrap around the end and continue from the start.

---

## Example 1

```java
Input: nums = [1,-2,3,-2]

Output: 3
```

### Explanation

Maximum subarray:

```text
[3]
```

Sum:

```text
3
```

Output:

```text
3
```

---

## Example 2

```java
Input: nums = [5,-3,5]

Output: 10
```

### Explanation

Normal maximum subarray:

```text
[5]
```

Sum = 5

Circular subarray:

```text
[5] + [5]
```

Wrap around:

```text
5 + 5 = 10
```

Output:

```text
10
```



### Explanation

Maximum element:

```text
-2
```

Output:

```text
-2
```

# Understanding Circular Arrays

For:

```java
nums = [5,-3,5]
```

Normal view:

```text
5 → -3 → 5
```

Circular view:

```text
5 → -3 → 5
↑         ↓
└─────────┘
```

Possible circular subarray:

```text
Last 5 + First 5
```

Sum:

```text
10
```

---

# Key Observation

The answer can be one of two cases:

## Case 1: Normal Maximum Subarray

Use Kadane's Algorithm.

Example:

```java
[1,-2,3,-2]
```

Maximum:

```text
[3]
```

Sum:

```text
3
```

---

## Case 2: Circular Maximum Subarray

Imagine removing the minimum-sum subarray.

Then:

```text
Remaining elements form the maximum circular subarray.
```

Formula:

MaximumCircularSum=TotalSum-MinimumSubarraySum

---

# Example

```java
nums = [5,-3,5]
```

Total Sum:

```text
5 + (-3) + 5 = 7
```

Minimum Subarray:

```text
[-3]
```

Minimum Sum:

```text
-3
```

Circular Sum:

```text
7 - (-3)

= 10
```

Answer:

```text
10
```

---

# Algorithm

1. Find maximum subarray sum (Kadane).
2. Find minimum subarray sum.
3. Find total array sum.
4. Calculate:

```text
normalMax
```

and

```text
totalSum - minSubarray
```

5. Return the larger value.

---

# Important Edge Case

Consider:

```java
nums = [-3,-2,-3]
```

Maximum Subarray:

```text
-2
```

Minimum Subarray:

```text
Entire Array
```

Total Sum:

```text
-8
```

Circular Sum:

```text
-8 - (-8)

= 0
```

This is invalid because it means taking no elements.

So:

```text
If normalMax < 0
return normalMax
```

# Java Solution (Optimal)

```java
class Solution {
    public int maxSubarraySumCircular(int[] nums) {

        int totalSum = 0;

        int currentMax = 0;
        int maxSum = nums[0];

        int currentMin = 0;
        int minSum = nums[0];

        for (int num : nums) {

            totalSum += num;

            currentMax = Math.max(
                    num,
                    currentMax + num
            );

            maxSum = Math.max(
                    maxSum,
                    currentMax
            );

            currentMin = Math.min(
                    num,
                    currentMin + num
            );

            minSum = Math.min(
                    minSum,
                    currentMin
            );
        }

        if (maxSum < 0) {
            return maxSum;
        }

        return Math.max(
                maxSum,
                totalSum - minSum
        );
    }
}
```

| Operation | Complexity |

---

# Why Does `totalSum - minSum` Work?

Suppose:

```java
nums = [5,-3,5]
```

Removing:

```text
[-3]
```

Leaves:

```text
[5] and [5]
```

which connect in a circular array.

Therefore:

```text
Maximum Circular Sum

=

Total Sum

-

Minimum Subarray Sum
```


**Time Complexity:** O(n)
**Space Complexity:** O(1)**
