## 📌 Problem Statement

You are given an array of strings `tokens` representing an arithmetic expression in **Reverse Polish Notation (RPN)**.

Return the value of the expression.

The valid operators are:

```
+
-
*
/
```

Rules:

- Every operand is either an integer or another expression.
- Division truncates toward zero.
- There is no division by zero.
- The expression is always valid.

---

# 🧩 What is Reverse Polish Notation?

In normal mathematics (Infix Expression):

```text
2 + 3
```

Operator comes **between** operands.

---

In Reverse Polish Notation (Postfix):

```text
2 3 +
```

Operator comes **after** operands.

Another example

Normal:

```text
(2 + 1) × 3
```

RPN:

```text
2 1 + 3 *
```

---

# ✅ Java Implementation

```java
import java.util.Stack;

class Solution {

    public int evalRPN(String[] tokens) {

        Stack<Integer> st = new Stack<>();

        for(String ch : tokens){

            if(ch.equals("+")){

                st.push(st.pop() + st.pop());

            }

            else if(ch.equals("-")){

                int second = st.pop();
                int first = st.pop();

                st.push(first - second);

            }

            else if(ch.equals("*")){

                st.push(st.pop() * st.pop());

            }

            else if(ch.equals("/")){

                int second = st.pop();
                int first = st.pop();

                st.push(first / second);

            }

            else{

                st.push(Integer.parseInt(ch));

            }

        }

        return st.peek();

    }

}
```

## Space Complexity
