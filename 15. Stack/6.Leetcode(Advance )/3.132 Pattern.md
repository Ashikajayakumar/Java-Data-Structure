# LeetCode 456: 132 Pattern

## Problem Statement

Given an integer array `nums`, find whether there exists a **132 pattern**.

A **132 pattern** means:

```
nums[i] < nums[k] < nums[j]

where:

i < j < k
```

Meaning:

```
1        3        2
|        |        |
small   large   middle
```

The indices must satisfy:

```
i comes before j comes before k
```

---

# Example

## Input

```java
nums = [3,1,4,2]
```

We need to find:

```
1 3 2
```

Look at:

```
index:  0  1  2  3

nums:   3  1  4  2
```

Choose:

```
i = 1
j = 2
k = 3
```

Values:

```
nums[i] = 1
nums[j] = 4
nums[k] = 2
```

Check:

```
nums[i] < nums[k] < nums[j]

1 < 2 < 4
```

True.

So answer:

```
true
```

---

# Brute Force Approach

We can try three loops:

```java
for(i)
    for(j)
        for(k)
```

Check:

```
nums[i] < nums[k] < nums[j]
```

Time:

```
O(N^3)
```

Too slow.

---

# Optimized Idea: Monotonic Stack

We scan from **right to left**.

Why?

Because in a 132 pattern:

```
1 3 2

3 must be before 2
```

When scanning from right side:

We can easily find:

```
2 (third value)
```

and compare it with possible:

```
1
```

# Complete Code Explanation

```java
class Solution {

    public boolean find132pattern(int[] nums) {


        Stack<Integer> st = new Stack<>();


        // Stores the "2" value
        int third = Integer.MIN_VALUE;



        // Scan from right to left
        for(int i = nums.length-1; i >=0; i--){



            // Found nums[i] < nums[k]
            if(nums[i] < third){

                return true;

            }



            // Find possible middle value
            while(!st.isEmpty() && st.peek() < nums[i]){


                third = st.pop();


            }



            // Store possible 3 value
            st.push(nums[i]);


        }


        return false;
    }
}
```
